## 应急修复记录

- 现象：训练后期出现上游 embedding `400 The parameter is invalid`，并且 Mongo `indexes.dataId` 引用与 pgvector `modeldata` 行数未完全对齐。
- 错误尝试：曾尝试清理 Mongo `indexes.dataId` 触发 schema 校验失败，已通过 `/tmp/force-rebuild-mongo.archive.gz` 恢复 Mongo。
- 最终方案：保留 Mongo `dataset_datas.indexes.dataId` 不变，以 Mongo 中现有索引引用为准，原地重建 pgvector `modeldata`。
- 特殊数据：一条超长 Clash 配置表索引触发 provider 参数错误，使用紧凑摘要生成向量，正文内容未删除。
- 最终状态：
  - Mongo `dataset_datas=8205`
  - Mongo `dataset_trainings=0`
  - Mongo `error_tasks=0`
  - Mongo unique vector refs `uniqueVectorIds=6919`
  - Postgres `modeldata_count=6919`
  - Postgres `distinct_ids=6919`
  - Postgres vector dims `min_dims=2048`，`max_dims=2048`
- 固化措施：
  - 新增维护脚本 `ops/scripts/server/rebuild-pgvector-from-mongo-indexes.sh`，用于按 Mongo `indexes.dataId` 安全重建 pgvector。

## 备注

- pgvector HNSW 普通 `VECTOR` 超过 2000 维会报错，因此生产采用 `HALFVEC(2048)`。该方案保留 2048 维检索能力，同时支持 HNSW 索引。
- 本次迁移会清空旧向量索引并触发全量向量重建；正文数据 `dataset_datas` 保留。
- `url-directory-005` 仍使用本地构建上传部署，原因是本次 GitHub token 暂不具备修改 workflow 的 `workflow` scope。
