# 未实现功能解决方案

## 结论

当前项目已经从 FastGPT 上游项目转为私有二次开发基线，后续不默认同步上游。项目需求优先通过修改本地 FastGPT 代码实现；原 FastGPT 中暂时不能用、依赖付费授权、外部服务或重资源组件的功能先隐藏入口，避免用户误操作。

## 处理原则

- 能用本地代码补齐的功能，优先实现。
- 依赖模型渠道的功能，不在代码中写死模型，继续通过 FastGPT 原有模型配置页面维护。
- 依赖 sandbox、MCP server、volume-manager、商业账单、工单、优惠券、模板市场等当前不可用能力的入口先隐藏。
- 不删除 FastGPT 原始能力代码，先用 feature flag 和路由守卫控制，便于后续按需恢复或接通。
- 不提交、不部署，除非用户明确要求。

## 已纳入本地代码处理的隐藏项

FastGPT 本地源码已增加集中式功能开关和入口控制：

- 隐藏 GitHub/star、官方文档、插件提交、模板课程等外部 FastGPT 官方入口。
- 隐藏付费、账单、优惠券、推广、自定义域名入口。
- 隐藏模板市场、系统工具市场、MCP server 管理、评测入口。
- 隐藏 Feishu、Yuque、DingTalk 数据集导入和 Feishu、DingTalk、WeCom、公众号、微信发布入口。
- 隐藏 Agent sandbox 入口和数据集增强入口。
- 保留模型提供商、模型渠道、知识库、应用、HTTP 工具、API Key 等当前核心可用功能。

## 未实现功能方案

| 功能 | 当前状态 | 优先方案 | 验收标准 |
| --- | --- | --- | --- |
| URL 目录导入 | 已有本地二开基础能力 | 继续内嵌在 FastGPT 知识库导入流程，不做独立项目页面 | 知识库内可选择 URL 目录导入、预览、过滤、开始导入、查看失败明细 |
| 多策略 URL 发现 | 部分实现 | 按 sitemap、首页导航、分类页、归档页、文章链接规则、手动补充 URL 分阶段实现 | 无 sitemap 站点也能发现主要文章 URL，并排除首页、分类、归档、侧边栏、广告 |
| URL 导入安全 | 待补强 | 在预览和导入接口增加 SSRF 防护、跳转限制、超时、大小限制 | 禁止 localhost、内网 IP、metadata IP；异常 URL 不进入导入任务 |
| 可点击内容来源 | 部分实现 | 统一回答来源格式为 `[来源标题](来源URL)`，必要时在前端或 prompt 层规范化 | 每条来源标题本身可点击，不再出现标题后跟裸 URL 的混合格式 |
| 来源列表页 | 待实现 | 在知识库/集合详情增加来源清单，按 title、url、type、collection 展示 | 可列出全部内容来源，支持搜索、筛选、点击跳转 |
| 原始资料归档 | 部分依赖 FastGPT storage | 使用 FastGPT 现有文件存储为主，补 metadata 记录 title/source_type/original_url/original_path/tags | 上传和 URL 导入后能追溯原始文件或原始链接 |
| 动态分类推荐 | 待实现 | 先用知识库目录/标签手动维护，再增加 AI 推荐分类 API 和确认弹窗 | 新资料入库前可给出 Top 分类建议，用户确认后保存 |
| 多类型资料验收 | 待系统验证 | 使用 fixtures 验证 PDF、DOCX、TXT、Markdown、手动文本、URL | 每类资料可入库、向量化、问答、引用来源 |
| 备份脚本 | 待实现 | 增加 MongoDB、PostgreSQL、storage 备份脚本和保留策略 | 可一键生成备份，恢复流程有文档 |
| 资源优化 | 已有方向 | 不使用本地 embedding；降低 worker/process 数；仅开启必要容器 | 2C/4G 下内存长期低于 3.5GB，知识库服务稳定响应 |
| sandbox/MCP/volume-manager | 当前不作为必要功能 | 资源允许且有明确用例时再开启；否则保持隐藏 | 开启前有资源评估、回滚方案和功能验收 |

## 分阶段执行

### 阶段 1：当前可用闭环

- 保持模型配置走 FastGPT 原有页面。
- 保持本地 embedding 关闭。
- 完成 URL 目录导入、来源可点击、来源清单、基础资料入库验收。
- 隐藏不可用 FastGPT 原功能入口。

### 阶段 2：安全和质量补强

- 补 URL 导入 SSRF 防护。
- 补多策略 URL 发现。
- 补 metadata 归档和来源列表。
- 补备份脚本。

### 阶段 3：智能化增强

- 增加动态分类推荐。
- 增加标签建议。
- 增加导入质量检测和失败重试。
- 按资源状态评估是否开启 sandbox、MCP、volume-manager。

## 版本维护规则

- FastGPT 源码仓库作为私有二开仓库维护。
- 不默认 pull、merge 或 rebase FastGPT 上游。
- 后续升级只基于本地产品需求和明确版本任务执行。
- 每次部署前必须记录 FastGPT 本地 commit、ai-knowledge-base 文档 commit、镜像 tag、服务器快照和回滚目标。
