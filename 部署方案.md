# AI 知识库一步到位实施方案

## 结论

采用 FastGPT 官方 Docker Compose `PgVector` 轻量部署方案作为基础。本地先完成 V1 闭环验证，验证通过后把同一套配置迁移到腾讯云服务器，由 Nginx 按域名分流：

- `api.zero007.chat` -> 现有 sub2api
- `kb.zero007.chat` -> FastGPT

`kb.zero007.chat` 继续在 Cloudflare 配置 DNS，不需要回腾讯云 DNS 配置。两个域名可以指向同一台服务器，不会冲突。

## 本地阶段

1. 从官方地址下载 FastGPT PgVector Docker Compose 模板。
2. 从 `key.md` 读取 sub2api `baseUrl` 和 `API key`，生成本地私有 `.env.local`。
3. 自动检测端口占用，默认优先使用：
   - FastGPT Web: `3000`
   - AIProxy: `3001`
   - PostgreSQL: `5432`
   - Redis: `6379`
   - MongoDB: `27017`
4. 端口被占用时自动切换到后续可用端口。
5. 启动 Docker Compose。
6. 验证：
   - 容器状态
   - FastGPT Web 可访问
   - Chat 模型可调用
   - Embedding 模型可调用
7. 生成 PDF / DOCX / TXT / Markdown 测试资料和验收问题。

## 服务器阶段

1. 通过 `ssh sub2api` 登录服务器。
2. 创建目录：
   - `/opt/fastgpt`
   - `/opt/fastgpt/config`
   - `/opt/fastgpt/logs`
   - `/storage/docs`
   - `/storage/images`
   - `/storage/web`
   - `/storage/videos`
   - `/storage/derived`
   - `/storage/tmp`
3. 上传本地验证通过的 Compose 和配置。
4. 启动 FastGPT / MongoDB / PostgreSQL + pgvector / Redis / AIProxy。
5. 配置 Nginx：
   - `kb.zero007.chat` 反代 FastGPT
   - `api.zero007.chat` 保持现有 sub2api
6. 配置 HTTPS 和自动续期。
7. 完成上传、向量化、问答、引用和原文查看验收。

## GitHub 仓库方案

建议创建私有仓库：

```text
https://github.com/zeroliao/ai-knowledge-base
```

本机当前未检测到 `gh` CLI。可选路径：

1. 安装并登录 `gh`，由 Codex 创建仓库并 push。
2. 用户手动创建私有仓库，Codex 绑定 remote 并 push。

无论哪种方式，都不会提交：

- `key.md`
- `.env`
- `.env.local`
- 密码
- API Key
- SSH private key

## Cloudflare 配置

在 Cloudflare 的 `zero007.chat` DNS 中新增：

```text
Type: A
Name: kb
Content: 腾讯云服务器公网 IP
Proxy status: DNS only
TTL: Auto
```

稳定后可按需开启 Cloudflare Proxy。首次部署建议 `DNS only`，减少 HTTPS 和 WebSocket 排障变量。

## 安全策略

- FastGPT 管理后台登录后访问。
- 原文查看必须经过应用鉴权。
- `/storage` 不通过 Nginx 直接裸露。
- `key.md` 和所有 `.env*` 不提交 GitHub。
- 上线后建议把初始管理员密码改成强密码。
- URL 抓取进入 V2 后必须加入 SSRF 防护。
